# ConverTool v1.0.2

本版本在 **v1.0.1** 基础上增强**图片插件的目标体积累计压缩**、**输出文件名的标签式编排**，并修复若干问题。  
**版本对齐**：Host、`ConverTool.PluginAbstractions` NuGet、仓库内自带插件的 `manifest.json` **`version`** 均为 **1.0.2**（见 [`docs/plugin-dev.md`](../plugin-dev.md) §3.1）。

---

## 面向普通用户

### 图片：按目标大小压缩

- 在 **ImageMagick 图片转换** 插件中，可开启**按目标文件大小**进行压缩（有损格式，如 JPEG / WebP 等）。
- 可设置**目标体积区间**（支持 KB / MB），程序会通过调节质量等方式，尽量让输出文件落在该区间内。
- **说明**：无损格式（如 PNG）无法通过「压质量」满足体积累计目标时，插件会按实现策略提示或跳过；具体以插件界面与日志为准。

### 输出：用标签拼文件名，并支持日期时间

- **输出命名**改为**标签（芯片）**方式：从候选区点选 **文件名**、**序号**、**日期**、**时分秒** 等，组合成模板；已选标签可**拖动排序**。
- 多个标签之间用 **下划线 `_`** 连接；右侧仍会显示当前**目标格式的扩展名**（与 v1.0.1 起行为一致）。
- 新增时间类占位：**年月日**（`yyyy-MM-dd`）、**时分秒**（`HH-mm-ss`），便于批量导出时区分生成时间。

### 其他

- 修复若干界面与流程上的问题，提升稳定性（详见下文「面向开发者 / 技术说明」）。

## 面向开发者 / 技术说明

### Host

- **命名模板**
  - UI：`NamingTemplateSelectedTags` + 拖拽重排（`MainWindow.axaml` / `MainWindow.axaml.cs`）。
  - 持久化字符串由 `string.Join("_", tags)` 生成；落盘时在 `MoveToFinalOutput` 中替换 **`{base}`、`{ext}`、`{index}`、`{timeYmd}`、`{timeHms}`**（时间与界面文案一致：`yyyy-MM-dd`、`HH-mm-ss`）。
  - `ExecuteContext.OutputNamingContext` 同步提供键 **`base`、`index`、`ext`、`timeYmd`、`timeHms`**（字符串/数值与 Host 约定一致），供插件只读使用。
- **ImageMagick 插件（`plugins-src/ImageMagickImageTranscoder`）**
  - 目标体积累计压缩：`enableTargetSizeCompression`、`targetSizeMinKb` / `targetSizeMaxKb`、单位字段；对质量做二分搜索等逻辑见源码与 `manifest.json`。
  - 与 **`fieldPersistOverrides`** 等配合，处理「保留设置」与持久化快照（与契约 **1.0.1+** 一致）。

### 契约与插件

- **`ConverTool.PluginAbstractions` 1.0.2**：`ConfigFieldType` 等与 JSON manifest 对齐；**`MultiSelect`** 已在契约枚举中定义，**Host v1.0.2 仍未渲染该类型**，插件 manifest 中请勿使用（见 [`plugin-dev.md`](../plugin-dev.md)）。
- 独立契约仓库发版说明见该仓库 **`README.md`** / **`CHANGELOG.md`**。

## 升级说明

- 自 **v1.0.1** 升级：直接覆盖安装或解压新版即可；用户设置与插件目录逻辑不变。
- 若曾手动编辑过命名模板字符串，升级后会按新规则解析为标签；空模板会回退为安全默认。
