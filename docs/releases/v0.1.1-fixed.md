# ConverTool Host v0.1.1-fixed

基于 **v0.1.1** 的修订版，主要修复交互稳定性与 Range 配置区布局。

## 变更摘要

### 修复：滑块导致无响应 / 崩溃

- **原因**：全局主题 `Win11.axaml` 曾为所有 `Slider` 的 `Value` 配置 `DoubleTransition`。在 Avalonia 中，`Slider.Value` **双向绑定**与 **Value 过渡动画** 容易形成反馈回路，表现为拖动滑块后界面卡死、内存飙升或直接退出。
- **处理**：移除针对 `Slider.Value` 的 `DoubleTransition`（并行度滑块与插件 `Range` 滑块均受益）。

### 修复：插件 `Range` 配置在宽屏下轨道过长

- **原因**：`MainWindow.axaml` 中 Range 行使用 `Grid` 列 `*,100`，滑块放在 `*` 列会横向拉伸填满配置区剩余宽度，配置区越宽轨道越长（与插件 `manifest` 的 `range` 无关）。
- **处理**：改为 `Auto,*,80`：滑块左对齐并设置 **`MinWidth="120"` / `MaxWidth="280"`**，中间列吸收多余空间，数字输入框固定在右侧。

### 其他（v0.1.1 起已包含，本版一并说明）

- **Range 字段**：当 `(max - min) / step` 过大时，避免 `IsSnapToTickEnabled` + 过小 `TickFrequency` 导致 UI 极慢；自动关闭吸附并放宽 `TickFrequency`（仍可通过右侧文本框输入精确数值）。
